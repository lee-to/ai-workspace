# Align implementation with research spec (session 2)

**Date:** 2026-03-16 17:30
**Files:** src/db/schema.rs, src/models.rs, src/db/crud.rs, src/db/mod.rs, src/cli/mod.rs, src/mcp/protocol.rs, src/mcp/tools.rs, tests/cli_tests.rs, tests/mcp_tests.rs
**Severity:** medium

## Problem

Verification found 9 gaps between the research spec and the implementation. The data model was missing `dir` kind, `label` field, `updated_at`, `created_by_project_id`, and project-level notes. CLI commands had simplified interfaces (rm by ID only, no --label, no --scope). FTS5 indexed only `content` instead of `label+content`. 5 compiler warnings from unused code.

## Root Cause

The research spec evolved in session 2 (adding dir kind, labels, project notes, created_by_project_id, updated_at, rm by label/path) but the initial implementation was built against the simpler session-1 model. The plan tasks were high-level (e.g., "Database layer: schema, connection, CRUD") without explicit acceptance criteria referencing the session-2 decisions, so the implementation matched the letter of the task descriptions but not the full research context.

## Solution

Bottom-up cascade across 9 files:
1. **Schema**: Added `label`, `updated_at`, `created_by_project_id` columns; expanded `kind` CHECK to include `dir`; rewrote CHECK constraint for 4 patterns (file, dir, project note, group note); rebuilt FTS5 to index `label+content`
2. **Models**: Added `Dir` variant to `SharedItemKind`; added 3 new fields to `SharedItem`; removed unused `ProjectGroup` struct
3. **CRUD**: Split `add_note` into `add_project_note`/`add_group_note`; added `share_dir`; added `remove_by_label`/`remove_by_path`; updated all SELECT queries for new columns; updated `sync_files` for dir kind
4. **CLI**: Added `--label` to share/note; added `--scope` to note; changed `rm` to accept ID/label/path; added `NoteScope` enum
5. **MCP tools**: Updated workspace_context to include labels; used `invalid_params` for proper JSON-RPC errors; added dir listing in workspace_read
6. **Warnings**: Removed unused exports from db/mod.rs; suppressed dead_code on protocol.rs fields; removed unused `run_cmd` from tests
7. **Tests**: Added 4 new tests (share_directory, note_project_scope, rm_by_label, rm_by_path); updated existing tests for new interfaces; verified labels in MCP output

## Prevention

- When planning implementation from research, explicitly cross-reference ALL decisions from the research summary — not just the task titles
- Include acceptance criteria in plan tasks that reference specific field names and CLI flags
- Run `/aif-verify` immediately after implementation to catch spec drift early
- Consider generating a "delta checklist" between research decisions and implementation during planning
